Little Coders

As part of a 3rd year Software Engineering design course, our cohort had to work in groups of eight to develop a game using the Unity engine. The brief we were given for this project was fairly vague, other than that the game should address issues of diversity in Software Engineering. The game my team came up with was an idle-simulation game, where the player acts the role of a manager of a software startup.

The gameplay consists of two cycling phases. The first is where the player completes projects. Each project takes a short time to complete, and the player receives a monetary score upon each project's completion. This score depends on the happiness of your employees.

The second phase of gameplay is hiring. Hiring is essential to progress through the game, and occurs once the player has enough money to hire new employees. In this game, it is important to hire diversely. If the player consistently hires employees of the same demographic, the employees will start to complain, causing the monetary score of each project to decrease, and possibly even causing some employees to leave due to poor workplace culture.

This project was very interesting to work on, as we got to be very creative (all the game assets were designed and created by the team). It was also my first experience working in a shared repository with so many people, so we had to come up with processes to manage our git workflow.

This project won the best game award out of all the projects submitted by the cohort.

Tātai!

Tātai! was a project that all part II Software Engineers had to complete in groups of two. We were given a short project brief detailing the main functionality required, and were left to build the application as we saw fit. Having no skeleton code, we had to think very carefully about the overall system design and how we would coordinate our code. We decided our target users were to be young Māori children, who would be using the app to improve their basic maths.

This was my first experience programming in a team, and consequently I learnt a lot from this project.

MIDIsuite

I've always wanted to work on a project that in some way helps my piano channel - Datty Plays. The natural way to combine a programming project with piano is to make use of MIDI files, which are a digital representation of a musical performance. Something that I thought would be really cool would be to be able to generate visualisations of my piano performances, that I could then upload to YouTube in accompaniment to the audio, rather than just a video of me playing. This project was heavily inspired by the smalin YouTube channel.

The finished project works by taking a midi file and an image as input, and generates a video of animated notes scrolling along the screen. When a note hits the vertical bar in the center of the screen, it's corresponding sound is played. Each piano piece produces a very different looking visualisation, and the finished product has the ability to customise the note colors, opacity, and scrolling speed. I am very happy with how this project turned out.

One function I am especially proud of, and hence am mentioning in this write-up, is the export functionality. Late in 2018 I had a more-or-less finished product, where a visualisation would be played, but I had no way to access this visualisation as a video file that I could then upload to YouTube. So clearly I needed to write some code to export this visualisation to a video format.

Implementing this export functionality was far more difficult than expected, as I was using a 3rd party library to create the animations which in itself didn't provide an export option. After extensive research and a lot of proofs of concepts, I managed to get the functionality working by capturing the content of the html canvas element every 1/30th of a second, and writing each frame out to disk as a png. When the number of png's on disk reaches a certain threshold, I make a call to ffmpeg to combine all the pngs into a video at 30 frames per second. This worked like a charm, as you can see on my more recent YouTube videos.
