Website 2.0

At my Datacom internship over Summer '17/'18 I gained a lot of exposure to modern web development tools and workflows. All this new information made me question why I built this website (the one you're on) the way I did. I am currently working on a new website, making it more professional than personal, and focusing on using modern tools to improve the development process and ensure my web development skills are up to date with the ever changing industry.

I am planning on using:

  • Webpack - so I can separate my JS files during development but deploy a single script.
  • Bootstrap - for responsiveness.
  • React.js - seems to be the best front-end framework for this site's purposes.
  • Gulp - for running build tasks. Allows me to separate concerns during development more easily.
This site will stay live and I will use it for more personal and/or fun content like my piano videos, and web apps.

Tātai!

Tātai! was a project that all part II Software Engineers had to complete in groups of two. We were given a short project brief detailing the main functionality required, and were left to build the application as we saw fit. Having no skeleton code, we had to think very carefully about the overall system design and how we would coordinate our code. We decided our target users were to be young Māori children, who would be using the app to improve their basic maths.

This was my first experience programming in a team, and consequently I learnt a lot from this project.

Arduino Project

One of my friends found an old car phone and thought it would be cool to program it using an Arduino, and use it as a sort of control panel in the car. For example it might display the current temperature and could control some of the car's electronics. My friend is relatively comfortable with the electrical side of things, and approached me to help with the programming part. This project is really ambitious but will probably be one of the most rewarding among the list of projects here.

MIDIsuite

I've always wanted to work on a project that in some way helps my piano channel - Datty Plays. The natural way to combine a programming project with piano is to make use of MIDI files, which are a digital representation of a musical performance. I am currently familiarising myself with the MIDI file format, which is essential to understand before I can do anything with MIDI. My end goal is to produce software that will automatically generate a visualisation of the MIDI performance.

This project is heavily inspired by the smalin YouTube channel.
